# Alibaba Java Coding Guidelines (P3C) - 项目精简版

本规范旨在通过限制过度的个性化，提升团队协作效率，并规避常见的 Java 编程陷阱（如 NPE、OOM、并发死锁等）。AI 在生成代码时必须严格遵循以下规则。

## 1. 命名风格 (Naming Style)

- **[强制]** 代码中的命名严禁使用拼音与英文混合的方式，更不允许直接使用中文。
- **[强制]** 类名使用 `UpperCamelCase` 风格，方法名、参数名、成员变量、局部变量都统一使用 `lowerCamelCase` 风格。
- **[强制]** 常量命名全部大写，单词间用下划线隔开，力求语义表达完整清楚，不要嫌名字长。
- **[强制]** 抽象类命名使用 `Abstract` 或 `Base` 开头；异常类命名使用 `Exception` 结尾；测试类命名以它要测试的类名开始，以 `Test` 结尾。

## 2. OOP 规约 (OOP Rules)

- **[强制]** 所有的覆写方法，必须加 `@Override` 注解。
- **[强制]** POJO 类属性、RPC 方法返回值/参数必须使用**包装数据类型**（如 `Integer` 而非 `int`），避免初始值（0）与“无数据”的歧义。
- **[强制]** POJO 类必须包含 `toString()`。使用 IDE 生成或使用 Lombok 的 `@Data`/`@ToString`。
- **[强制]** 禁止在 POJO 类中同时定义 `isXxx()` 和 `getXxx()` 形式的属性，避免混淆。

## 3. 集合处理 (Collections)

- **[强制]** 只要重写了 `equals`，就必须重写 `hashCode`。
- **[强制]** 不要在 `foreach` 循环里进行元素的 `remove`/`add` 操作。删除元素请使用 `Iterator`。
- **[强制]** 集合初始化时，指定初始容量（避免频繁扩容导致的性能损耗）。
  - `Map<String, String> map = new HashMap<>(16);`

## 4. 并发处理 (Concurrency)

- **[强制]** 线程资源必须通过线程池提供，不允许在应用中自行显式创建线程（禁止 `new Thread().start()`）。
- **[强制]** **禁止**使用 `Executors` 去创建线程池（规避 OOM 风险），必须通过 `ThreadPoolExecutor` 的方式手动指定核心参数。
- **[强制]** 必须回收自定义的 `ThreadLocal` 变量（显式调用 `remove()`），避免内存泄漏。
- **[强制]** 获取单例对象需要保证线程安全，推荐使用“静态内部类”或“枚举”方式实现。

## 5. 异常与日志 (Exceptions & Logging)

- **[强制]** 禁止捕获异常后“吞掉”（Catch 块为空或仅有一行 `e.printStackTrace()`）。至少要在日志中记录 `error` 级别信息。
- **[强制]** 级联调用（如 `obj.getA().getB().getC()`）极易触发 NPE，推荐使用 Java 8+ 的 `Optional` 处理。
- **[强制]** 在 `TRACE`/`DEBUG`/`INFO` 级别输出日志时，必须使用占位符 `{}`，禁止使用字符串拼接，避免不必要的对象创建开销。

## 6. MySQL 数据库 (Database)

- **[强制]** 表名、字段名必须使用小写字母或数字，禁止出现数字开头。
- **[强制]** 业务上具有唯一特性的字段，即使是组合字段，也必须建成唯一索引（Unique Index）。
- **[强制]** 禁止超过三个表的 `JOIN`。在业务层（Service）进行逻辑拼接往往比复杂的 SQL 更具可扩展性。
- **[强制]** 在 `WHERE` 条件中，禁止对字段进行函数运算或表达式运算，否则会导致索引失效。
